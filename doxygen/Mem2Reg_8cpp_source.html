24 /// mem2reg
25 ///
26 /// This pass turns unnecessary uses of automatically allocated memory slots
27 /// into direct Value-based operations. For example, it will simplify storing a
28 /// constant in a memory slot to immediately load it to a direct use of that
29 /// constant. In other words, given a memory slot addressed by a non-aliased
30 /// "pointer" Value, mem2reg removes all the uses of that pointer.
31 ///
32 /// Within a block, this is done by following the chain of stores and loads of
33 /// the slot and replacing the results of loads with the values previously
34 /// stored. If a load happens before any other store, a poison value is used
35 /// instead.
36 ///
37 /// Control flow can create situations where a load could be replaced by
38 /// multiple possible stores depending on the control flow path taken. As a
39 /// result, this pass must introduce new block arguments in some blocks to
40 /// accomodate for the multiple possible definitions. Each predecessor will
41 /// populate the block argument with the definition reached at its end. With
42 /// this, the value stored can be well defined at block boundaries, allowing
43 /// the propagation of replacement through blocks.
44 ///
45 /// This pass computes this transformation in four main steps:
46 /// - A first step computes the list of operations that transitively use the
47 /// memory slot we would like to promote. The purpose of this phase is to
48 /// identify which uses must be removed to promote the slot, either by rewiring
49 /// the user or deleting it. Naturally, direct uses of the slot must be removed.
50 /// Sometimes additional uses must also be removed: this is notably the case
51 /// when a direct user of the slot cannot rewire its use and must delete itself,
52 /// and thus must make its users no longer use it. If any of those uses cannot
53 /// be removed by their users in any way, promotion cannot continue: this is
54 /// decided at this step.
55 /// - A second step computes the list of blocks where a block argument will be
56 /// needed ("merge points") without mutating the IR. These blocks are the blocks
57 /// leading to a definition clash between two predecessors. Such blocks happen
58 /// to be the Iterated Dominance Frontier (IDF) of the set of blocks containing
59 /// a store, as they represent the point where a clear defining dominator stops
60 /// existing. Computing this information in advance allows making sure the
61 /// terminators that will forward values are capable of doing so (inability to
62 /// do so aborts promotion at this step).
63 /// - A third step computes the reaching definition of the memory slot at each
64 /// blocking user. This is the core of the mem2reg algorithm, also known as
65 /// load-store forwarding. This analyses loads and stores and propagates which
66 /// value must be stored in the slot at each blocking user. This is achieved by
67 /// doing a depth-first walk of the dominator tree of the function. This is
68 /// sufficient because the reaching definition at the beginning of a block is
69 /// either its new block argument if it is a merge block, or the definition
70 /// reaching the end of its immediate dominator (parent in the dominator tree).
71 /// We can therefore propagate this information down the dominator tree to
72 /// proceed with renaming within blocks.
73 /// - The final fourth step uses the reaching definition to remove blocking uses
74 /// in topological order.
75 ///
76 /// The two first steps do not mutate IR because promotion can still be aborted
77 /// at this point. Once the two last steps are reached, promotion is guaranteed
78 /// to succeed, allowing to start mutating IR.
79 ///
80 /// For further reading, chapter three of SSA-based Compiler Design [1]
81 /// showcases SSA construction, where mem2reg is an adaptation of the same
82 /// process.
83 ///
84 /// [1]: Rastello F. & Bouchez Tichadou F., SSA-based Compiler Design (2022),
85 /// Springer.

89 /// The SlotPromoter handles the state of promoting a memory slot. It wraps a
90 /// slot and its associated allocator, along with analysis results related to
91 /// the slot.
92 class SlotPromoter {
93 public:
94  SlotPromoter(MemorySlot slot, PromotableAllocationOpInterface allocator,
95  OpBuilder &builder, DominanceInfo &dominance);
96 
97  /// Prepare data for the promotion of the slot while checking if it can be
98  /// promoted. Succeeds if the slot can be promoted. This method does not
99  /// mutate IR.
100  LogicalResult prepareSlotPromotion();
101 
102  /// Actually promotes the slot by mutating IR. This method must only be
103  /// called after a successful call to `SlotPromoter::prepareSlotPromotion`.
104  /// Promoting a slot does not invalidate the preparation of other slots.
105  void promoteSlot();
106 
107 private:
108  /// This is the first step of the promotion algorithm.
109  /// Computes the transitive uses of the slot that block promotion. This finds
110  /// uses that would block the promotion, checks that the operation has a
111  /// solution to remove the blocking use, and potentially forwards the analysis
112  /// if the operation needs further blocking uses resolved to resolve its own
113  /// uses (typically, removing its users because it will delete itself to
114  /// resolve its own blocking uses). This will fail if one of the transitive
115  /// users cannot remove a requested use, and should prevent promotion.
116  LogicalResult computeBlockingUses();
117 
118  /// Computes in which blocks the value stored in the slot is actually used,
119  /// meaning blocks leading to a load. This method uses `definingBlocks`, the
120  /// set of blocks containing a store to the slot (defining the value of the
121  /// slot).
122  SmallPtrSet<Block *, 16>
123  computeSlotLiveIn(SmallPtrSetImpl<Block *> &definingBlocks);
124 
125  /// This is the second step of the promotion algorithm.
126  /// Computes the points in which multiple re-definitions of the slot's value
127  /// (stores) may conflict.
128  void computeMergePoints();
129 
130  /// Ensures predecessors of merge points can properly provide their current
131  /// definition of the value stored in the slot to the merge point. This can
132  /// notably be an issue if the terminator used does not have the ability to
133  /// forward values through block operands.
134  bool areMergePointsUsable();
135 
136  /// Computes the reaching definition for all the operations that require
137  /// promotion. `reachingDef` is the value the slot should contain at the
138  /// beginning of the block. This method returns the reached definition at the
139  /// end of the block.
140  Value computeReachingDefInBlock(Block *block, Value reachingDef);
141 
142  /// This is the third step of the promotion algorithm.
143  /// Computes the reaching definition for all the operations that require
144  /// promotion. `reachingDef` corresponds to the initial value the
145  /// slot will contain before any write, typically a poison value.
146  void computeReachingDefInRegion(Region *region, Value reachingDef);
147 
148  /// This is the fourth step of the promotion algorithm.
149  /// Removes the blocking uses of the slot, in topological order.
150  void removeBlockingUses();
151 
152  /// Lazily-constructed default value representing the content of the slot when
153  /// no store has been executed. This function may mutate IR.
154  Value getLazyDefaultValue();
155 
156  MemorySlot slot;
157  PromotableAllocationOpInterface allocator;
158  OpBuilder &builder;
159  /// Potentially non-initialized default value. Use `lazyDefaultValue` to
160  /// initialize it on demand.
161  Value defaultValue;
162  /// Blocks where multiple definitions of the slot value clash.
163  SmallPtrSet<Block *, 8> mergePoints;
164  /// Contains, for each operation, which uses must be eliminated by promotion.
165  /// This is a DAG structure because an operation that must eliminate some of
166  /// its uses always comes from a request from an operation that must
167  /// eliminate some of its own uses.
168  DenseMap<Operation *, SmallPtrSet<OpOperand *, 4>> userToBlockingUses;
169  /// Contains the reaching definition at this operation. Reaching definitions
170  /// are only computed for promotable memory operations with blocking uses.
171  DenseMap<PromotableMemOpInterface, Value> reachingDefs;
172  DominanceInfo &dominance;
173 };

203 LogicalResult SlotPromoter::computeBlockingUses() {
204  // The promotion of an operation may require the promotion of further
205  // operations (typically, removing operations that use an operation that must
206  // delete itself). We thus need to start from the use of the slot pointer and
207  // propagate further requests through the forward slice.
208 
209  // First insert that all immediate users of the slot pointer must no longer
210  // use it.
211  for (OpOperand &use : slot.ptr.getUses()) {
212  SmallPtrSet<OpOperand *, 4> &blockingUses =
213  userToBlockingUses.getOrInsertDefault(use.getOwner());
214  blockingUses.insert(&use);
215  }
216 
217  // Then, propagate the requirements for the removal of uses. The
218  // topologically-sorted forward slice allows for all blocking uses of an
219  // operation to have been computed before we reach it. Operations are
220  // traversed in topological order of their uses, starting from the slot
221  // pointer.
222  SetVector<Operation *> forwardSlice;
223  mlir::getForwardSlice(slot.ptr, &forwardSlice);
224  for (Operation *user : forwardSlice) {
225  // If the next operation has no blocking uses, everything is fine.
226  if (!userToBlockingUses.contains(user))
227  continue;
228 
229  SmallPtrSet<OpOperand *, 4> &blockingUses = userToBlockingUses[user];
230 
231  SmallVector<OpOperand *> newBlockingUses;
232  // If the operation decides it cannot deal with removing the blocking uses,
233  // promotion must fail.
234  if (auto promotable = dyn_cast<PromotableOpInterface>(user)) {
235  if (!promotable.canUsesBeRemoved(slot, blockingUses, newBlockingUses))
236  return failure();
237  } else if (auto promotable = dyn_cast<PromotableMemOpInterface>(user)) {
238  if (!promotable.canUsesBeRemoved(slot, blockingUses, newBlockingUses))
239  return failure();
240  } else {
241  // An operation that has blocking uses must be promoted. If it is not
242  // promotable, promotion must fail.
243  return failure();
244  }
245 
246  // Then, register any new blocking uses for coming operations.
247  for (OpOperand *blockingUse : newBlockingUses) {
248  assert(llvm::is_contained(user->getResults(), blockingUse->get()));
249 
250  SmallPtrSetImpl<OpOperand *> &newUserBlockingUseSet =
251  userToBlockingUses.getOrInsertDefault(blockingUse->getOwner());
252  newUserBlockingUseSet.insert(blockingUse);
253  }
254  }
255 
256  // Because this pass currently only supports analysing the parent region of
257  // the slot pointer, if a promotable memory op that needs promotion is
258  // outside of this region, promotion must fail because it will be impossible
259  // to provide a valid `reachingDef` for it.
260  for (auto &[toPromote, _] : userToBlockingUses)
261  if (isa<PromotableMemOpInterface>(toPromote) &&
262  toPromote->getParentRegion() != slot.ptr.getParentRegion())
263  return failure();
264 
265  return success();
266 }

268 SmallPtrSet<Block *, 16>
269 SlotPromoter::computeSlotLiveIn(SmallPtrSetImpl<Block *> &definingBlocks) {
270  SmallPtrSet<Block *, 16> liveIn;
271 
272  // The worklist contains blocks in which it is known that the slot value is
273  // live-in. The further blocks where this value is live-in will be inferred
274  // from these.
275  SmallVector<Block *> liveInWorkList;
276 
277  // Blocks with a load before any other store to the slot are the starting
278  // points of the analysis. The slot value is definitely live-in in those
279  // blocks.
280  SmallPtrSet<Block *, 16> visited;
281  for (Operation *user : slot.ptr.getUsers()) {
282  if (visited.contains(user->getBlock()))
283  continue;
284  visited.insert(user->getBlock());
285 
286  for (Operation &op : user->getBlock()->getOperations()) {
287  if (auto memOp = dyn_cast<PromotableMemOpInterface>(op)) {
288  // If this operation loads the slot, it is loading from it before
289  // ever writing to it, so the value is live-in in this block.
290  if (memOp.loadsFrom(slot)) {
291  liveInWorkList.push_back(user->getBlock());
292  break;
293  }
294 
295  // If we store to the slot, further loads will see that value.
296  // Because we did not meet any load before, the value is not live-in.
297  if (memOp.getStored(slot))
298  break;
299  }
300  }
301  }
302 
303  // The information is then propagated to the predecessors until a def site
304  // (store) is found.
305  while (!liveInWorkList.empty()) {
306  Block *liveInBlock = liveInWorkList.pop_back_val();
307 
308  if (!liveIn.insert(liveInBlock).second)
309  continue;
310 
311  // If a predecessor is a defining block, either:
312  // - It has a load before its first store, in which case it is live-in but
313  // has already been processed in the initialisation step.
314  // - It has a store before any load, in which case it is not live-in.
315  // We can thus at this stage insert to the worklist only predecessors that
316  // are not defining blocks.
317  for (Block *pred : liveInBlock->getPredecessors())
318  if (!definingBlocks.contains(pred))
319  liveInWorkList.push_back(pred);
320  }
321 
322  return liveIn;
323 }

325 using IDFCalculator = llvm::IDFCalculatorBase<Block, false>;
326 void SlotPromoter::computeMergePoints() {
327  if (slot.ptr.getParentRegion()->hasOneBlock())
328  return;
329 
330  IDFCalculator idfCalculator(dominance.getDomTree(slot.ptr.getParentRegion()));
331 
332  SmallPtrSet<Block *, 16> definingBlocks;
333  for (Operation *user : slot.ptr.getUsers())
334  if (auto storeOp = dyn_cast<PromotableMemOpInterface>(user))
335  if (storeOp.getStored(slot))
336  definingBlocks.insert(user->getBlock());
337 
338  idfCalculator.setDefiningBlocks(definingBlocks);
339 
340  SmallPtrSet<Block *, 16> liveIn = computeSlotLiveIn(definingBlocks);
341  idfCalculator.setLiveInBlocks(liveIn);
342 
343  SmallVector<Block *> mergePointsVec;
344  idfCalculator.calculate(mergePointsVec);
345 
346  mergePoints.insert(mergePointsVec.begin(), mergePointsVec.end());
347 }

358 Value SlotPromoter::computeReachingDefInBlock(Block *block, Value reachingDef) {
359  for (Operation &op : block->getOperations()) {
360  if (auto memOp = dyn_cast<PromotableMemOpInterface>(op)) {
361  if (userToBlockingUses.contains(memOp))
362  reachingDefs.insert({memOp, reachingDef});
363 
364  if (Value stored = memOp.getStored(slot))
365  reachingDef = stored;
366  }
367  }
368 
369  return reachingDef;
370 }

372 void SlotPromoter::computeReachingDefInRegion(Region *region,
373  Value reachingDef) {
374  if (region->hasOneBlock()) {
375  computeReachingDefInBlock(&region->front(), reachingDef);
376  return;
377  }
378 
379  struct DfsJob {
380  llvm::DomTreeNodeBase<Block> *block;
381  Value reachingDef;
382  };
383 
384  SmallVector<DfsJob> dfsStack;
385 
386  auto &domTree = dominance.getDomTree(slot.ptr.getParentRegion());
387 
388  dfsStack.emplace_back<DfsJob>(
389  {domTree.getNode(&region->front()), reachingDef});
390 
391  while (!dfsStack.empty()) {
392  DfsJob job = dfsStack.pop_back_val();
393  Block *block = job.block->getBlock();
394 
395  if (mergePoints.contains(block)) {
396  BlockArgument blockArgument =
397  block->addArgument(slot.elemType, slot.ptr.getLoc());
398  builder.setInsertionPointToStart(block);
399  allocator.handleBlockArgument(slot, blockArgument, builder);
400  job.reachingDef = blockArgument;
401  }
402 
403  job.reachingDef = computeReachingDefInBlock(block, job.reachingDef);
404 
405  if (auto terminator = dyn_cast<BranchOpInterface>(block->getTerminator())) {
406  for (BlockOperand &blockOperand : terminator->getBlockOperands()) {
407  if (mergePoints.contains(blockOperand.get())) {
408  if (!job.reachingDef)
409  job.reachingDef = getLazyDefaultValue();
410  terminator.getSuccessorOperands(blockOperand.getOperandNumber())
411  .append(job.reachingDef);
412  }
413  }
414  }
415 
416  for (auto *child : job.block->children())
417  dfsStack.emplace_back<DfsJob>({child, job.reachingDef});
418  }

421 void SlotPromoter::removeBlockingUses() {
422  llvm::SetVector<Operation *> usersToRemoveUses;
423  for (auto &user : llvm::make_first_range(userToBlockingUses))
424  usersToRemoveUses.insert(user);
425  SetVector<Operation *> sortedUsersToRemoveUses =
426  mlir::topologicalSort(usersToRemoveUses);
427 
428  llvm::SmallVector<Operation *> toErase;
429  for (Operation *toPromote : llvm::reverse(sortedUsersToRemoveUses)) {
430  if (auto toPromoteMemOp = dyn_cast<PromotableMemOpInterface>(toPromote)) {
431  Value reachingDef = reachingDefs.lookup(toPromoteMemOp);
432  // If no reaching definition is known, this use is outside the reach of
433  // the slot. The default value should thus be used.
434  if (!reachingDef)
435  reachingDef = getLazyDefaultValue();
436 
437  builder.setInsertionPointAfter(toPromote);
438  if (toPromoteMemOp.removeBlockingUses(slot, userToBlockingUses[toPromote],
439  builder, reachingDef) ==
440  DeletionKind::Delete)
441  toErase.push_back(toPromote);
442 
443  continue;
444  }
445 
446  auto toPromoteBasic = cast<PromotableOpInterface>(toPromote);
447  builder.setInsertionPointAfter(toPromote);
448  if (toPromoteBasic.removeBlockingUses(slot, userToBlockingUses[toPromote],
449  builder) == DeletionKind::Delete)
450  toErase.push_back(toPromote);
451  }
452 
453  for (Operation *toEraseOp : toErase)
454  toEraseOp->erase();
455 
456  assert(slot.ptr.use_empty() &&
457  "after promotion, the slot pointer should not be used anymore");
458 }

460 void SlotPromoter::promoteSlot() {
461  computeReachingDefInRegion(slot.ptr.getParentRegion(), {});
462 
463  // Now that reaching definitions are known, remove all users.
464  removeBlockingUses();
465 
466  // Update terminators in dead branches to forward default if they are
467  // succeeded by a merge points.
468  for (Block *mergePoint : mergePoints) {
469  for (BlockOperand &use : mergePoint->getUses()) {
470  auto user = cast<BranchOpInterface>(use.getOwner());
471  SuccessorOperands succOperands =
472  user.getSuccessorOperands(use.getOperandNumber());
473  assert(succOperands.size() == mergePoint->getNumArguments() ||
474  succOperands.size() + 1 == mergePoint->getNumArguments());
475  if (succOperands.size() + 1 == mergePoint->getNumArguments())
476  succOperands.append(getLazyDefaultValue());
477  }
478  }
479 
480  allocator.handlePromotionComplete(slot, defaultValue);
481 }

483 LogicalResult SlotPromoter::prepareSlotPromotion() {
484  // First, find the set of operations that will need to be changed for the
485  // promotion to happen. These operations need to resolve some of their uses,
486  // either by rewiring them or simply deleting themselves. If any of them
487  // cannot find a way to resolve their blocking uses, we abort the promotion.
488  if (failed(computeBlockingUses()))
489  return failure();
490 
491  // Then, compute blocks in which two or more definitions of the allocated
492  // variable may conflict. These blocks will need a new block argument to
493  // accomodate this.
494  computeMergePoints();
495 
496  // The slot can be promoted if the block arguments to be created can
497  // actually be populated with values, which may not be possible depending
498  // on their predecessors.
499  return success(areMergePointsUsable());
500 }

502 LogicalResult mlir::tryToPromoteMemorySlots(
503  ArrayRef<PromotableAllocationOpInterface> allocators, OpBuilder &builder,
504  DominanceInfo &dominance) {
505  // Actual promotion may invalidate the dominance analysis, so slot promotion
506  // is prepated in batches.
507  SmallVector<SlotPromoter> toPromote;
508  for (PromotableAllocationOpInterface allocator : allocators) {
509  for (MemorySlot slot : allocator.getPromotableSlots()) {
510  if (slot.ptr.use_empty())
511  continue;
512 
513  SlotPromoter promoter(slot, allocator, builder, dominance);
514  if (succeeded(promoter.prepareSlotPromotion()))
515  toPromote.emplace_back(std::move(promoter));
516  }
517  }
518 
519  for (SlotPromoter &promoter : toPromote)
520  promoter.promoteSlot();
521 
522  return success(!toPromote.empty());
523 }

527 struct Mem2Reg : impl::Mem2RegBase<Mem2Reg> {
528  void runOnOperation() override {
529  Operation *scopeOp = getOperation();
530  bool changed = false;
531 
532  for (Region &region : scopeOp->getRegions()) {
533  if (region.getBlocks().empty())
534  continue;
535 
536  OpBuilder builder(&region.front(), region.front().begin());
537 
538  // Promoting a slot can allow for further promotion of other slots,
539  // promotion is tried until no promotion succeeds.
540  while (true) {
541  DominanceInfo &dominance = getAnalysis<DominanceInfo>();
542 
543  SmallVector<PromotableAllocationOpInterface> allocators;
544  // Build a list of allocators to attempt to promote the slots of.
545  for (Block &block : region)
546  for (Operation &op : block.getOperations())
547  if (auto allocator = dyn_cast<PromotableAllocationOpInterface>(op))
548  allocators.emplace_back(allocator);
549 
550  // Attempt promoting until no promotion succeeds.
551  if (failed(tryToPromoteMemorySlots(allocators, builder, dominance)))
552  break;
553 
554  changed = true;
555  getAnalysisManager().invalidate({});
556  }
557  }
558 
559  if (!changed)
560  markAllAnalysesPreserved();
561  }
562 };
