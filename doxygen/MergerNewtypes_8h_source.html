35 /// Tensor identifiers.
36 ///
37 /// Semantically, tensor identifiers could be chosen to be anything;
38 /// but operationally, they must be chosen such that the `Merger`
39 /// and `GenericOpSparsifier` agree. Therefore, the numeric values of
40 /// tensor identifiers are chosen to be the `BlockArgument::getArgNumber`
41 /// of the value passed to `Merger::buildTensorExp`, which ranges from
42 /// zero to `linalg::GenericOp::getNumOperands` for the op passed to
43 /// `GenericOpSparsifier::matchAndRewrite`.
44 using TensorId = unsigned;
45 
46 //===----------------------------------------------------------------------===//
47 /// Loop identifiers.
48 ///
49 /// These identifiers serve as proxies for the `$dim` argument to
50 /// `linalg::IndexOp`, however the numerical value of a `LoopId` should
51 /// not necessarily be equated with the numerical value of the corresponding
52 /// `$dim` argument. The `$dim` arguments are De Bruijn indices: that
53 /// is, they identify the loop which binds the loop-variable by counting
54 /// the enclosing loops from innermost to outermost, starting from zero.
55 /// Whereas `LoopId` are considered to be arbitrary names for identifying
56 /// loops; since the `Merger` does not care about the actual ordering of
57 /// loops, and leaves it up to the `LoopEmitter` to specify the actual
58 /// loop ordering (`LoopOrd`).
59 ///
60 /// TODO: Despite the above claim that `$dim` and `LoopId` need not be
61 /// numerically equal, some code in the `Merger` class does equate them
62 /// (e.g., `buildTensorExp`). So we need to explicate the exact relationship
63 /// between `$dim`, `LoopId`, and `LoopOrd`; especially with regards to their
64 /// providence. If `LoopId` really is supposed to be equated with `$dim`,
65 /// then we should change the name to `LoopIdx` or similar, to capture the
66 /// fact that its numerical value is not invariant when entering/exiting
67 /// loops (unlike `TensorId`, `ExprId`, `LatPointId`, and `LatSetId` which
68 /// are invariant identifiers).
69 using LoopId = unsigned;
70 
71 //===----------------------------------------------------------------------===//
72 /// A compressed representation of `std::pair<TensorId, LoopId>`.
73 /// The compression scheme is such that this also serves as an index
74 /// into the bitvector stored in `LatPoint` (since that bitvector is
75 /// just the implementation for a set of `TensorLoopId` values).
76 using TensorLoopId = unsigned;
