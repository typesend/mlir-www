This macro helps minimize repetition of the printf-and-exit idiom, as well as ensuring that we print some additional output indicating where the error is coming from— to make it easier to determine whether some particular error is coming from the runtime library's code vs from somewhere else in the MLIR stack.

(Since that can be hard to determine without the stacktraces provided by assertion failures.)
